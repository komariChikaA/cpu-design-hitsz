# miniRV RTL 参数、接口与功能说明

本文解释当前 CPU 数据通路相关 RTL 的模块参数、输入输出接口、内部信号和主要逻辑块。这里所说的“函数”是指模块或逻辑块实现的功能；本工程 RTL 没有使用 Verilog 的 `function ... endfunction` 语法。

## 一、先理解 Verilog 中这些词

| 写法                       | 含义                                                                                              |
| -------------------------- | ------------------------------------------------------------------------------------------------- |
| `module ... endmodule`   | 一个硬件模块，相当于带端口的电路单元，不是按顺序调用的软件函数                                    |
| `input` / `output`     | 模块的输入、输出端口；方向以当前模块为参照                                                        |
| `wire`                   | 连线类型，通常由`assign`、模块输出或组合逻辑驱动                                                |
| `reg`                    | 可在`always` 块中赋值的变量类型；它不一定是寄存器，只有时钟触发的 `always` 才通常综合成寄存器 |
| `[31:0]`                 | 32 位总线，最高位编号 31，最低位编号 0                                                            |
| `parameter`              | 实例化时可修改的模块参数，例如乘法器位宽                                                          |
| `localparam`             | 仅模块内部使用的常量，外部不能覆盖                                                                |
| `assign`                 | 连续赋值，输入变化后组合逻辑输出随之变化                                                          |
| `always @(*)`            | 组合逻辑块，块内读取的任一信号变化都会重新计算                                                    |
| `always @(posedge clk)`  | 时序逻辑，只在时钟上升沿更新状态                                                                  |
| `or posedge rst`         | 高电平异步复位；复位上升沿不必等待时钟即可生效                                                    |
| `<=`                     | 非阻塞赋值，用于时序逻辑；同一个时钟沿的右值都按更新前状态计算                                    |
| `?:`                     | 条件选择器，通常综合成二选一多路器                                                                |
| `                          | signal`                                                                                           |
| `$signed(x)`             | 临时把位模式解释成有符号补码，用于算术比较或算术移位                                              |
| `` `include``              | 把公共宏定义文件包含进当前源文件                                                                  |
| `` `ifdef RUN_TRACE``      | 只有定义了`RUN_TRACE` 时才编译其中的调试信号                                                    |
| `/* verilator public */` | 告诉 Verilator 让该信号可被仿真测试环境直接访问                                                   |

## 二、总体模块关系

```text
cpu_top
├─ cpu_core       CPU 数据通路与控制
├─ Inst_ROM       指令存储器适配器
│  └─ IROM        Vivado Block RAM IP
└─ Data_RAM       数据存储器适配器
   └─ DRAM        Vivado Block RAM IP

cpu_core
├─ PC             保存当前 PC
├─ NPC            计算下一 PC 和 PC+4
├─ Controller     指令译码并产生控制信号
├─ RF             32 个整数寄存器的读写
├─ SEXT           立即数拼接与符号扩展
├─ ALU            普通运算、分支比较、乘除结果选择
│  ├─ U_mul       有符号乘法所用绝对值乘法器
│  ├─ U_mulu      无符号乘法器
│  ├─ U_div       有符号除法所用绝对值除法器
│  └─ U_divu      无符号除法器
├─ MREQ           生成访存地址、字节使能和对齐写数据
└─ MEXT           选择读回字节/半字并扩展
```

## 三、`cpu_top`：CPU、指令存储器和数据存储器连接层

文件：[`cpu_top.v`](../miniRV_basic/src/rtl/cpu_top.v)

### 外部接口

| 端口        | 方向/位宽 | 含义                     |
| ----------- | --------- | ------------------------ |
| `cpu_clk` | 输入 1 位 | CPU、IROM、DRAM 共用时钟 |
| `cpu_rst` | 输入 1 位 | 高电平有效复位           |

### 内部接口

| 信号                   | 方向        | 含义                                 |
| ---------------------- | ----------- | ------------------------------------ |
| `cpu2ic_rreq`        | CPU → IROM | 取指请求                             |
| `cpu2ic_addr[31:0]`  | CPU → IROM | 取指字节地址，通常等于 PC            |
| `ic2cpu_valid`       | IROM → CPU | 指令返回有效                         |
| `ic2cpu_inst[31:0]`  | IROM → CPU | 返回的 32 位指令                     |
| `cpu2dc_ren[3:0]`    | CPU → DRAM | 读请求/字节通道使能；非 0 表示读请求 |
| `cpu2dc_addr[31:0]`  | CPU → DRAM | 数据访问字节地址                     |
| `dc2cpu_valid`       | DRAM → CPU | 读数据有效                           |
| `dc2cpu_rdata[31:0]` | DRAM → CPU | 读回的一个 32 位字                   |
| `cpu2dc_wen[3:0]`    | CPU → DRAM | 四个字节 lane 的写使能               |
| `cpu2dc_wdata[31:0]` | CPU → DRAM | 已对齐到对应 lane 的写数据           |
| `dc2cpu_wresp`       | DRAM → CPU | 写请求完成应答                       |

`cpu_top` 本身不执行指令，只实例化三个模块并把同名功能接口连接起来。

## 四、`cpu_core`：整颗处理器核心

文件：[`cpu_core.v`](../miniRV_basic/src/rtl/cpu_core.v)

### 4.1 外部端口

| 端口                    | 方向/位宽  | 有效条件与含义                                  |
| ----------------------- | ---------- | ----------------------------------------------- |
| `cpu_rst`             | 输入 1 位  | 高电平异步复位核心状态                          |
| `cpu_clk`             | 输入 1 位  | 核心时钟                                        |
| `ifetch_req`          | 输出 1 位  | 请求读取`ifetch_addr` 所指指令                |
| `ifetch_addr[31:0]`   | 输出 32 位 | 取指地址，直接等于当前`pc`                    |
| `ifetch_valid`        | 输入 1 位  | 为 1 时`ifetch_inst` 有效且只保证一个时钟周期 |
| `ifetch_inst[31:0]`   | 输入 32 位 | 指令存储器返回的指令                            |
| `daccess_ren[3:0]`    | 输出 4 位  | 数据读请求；本设计合法读请求通常为`1111`      |
| `daccess_addr[31:0]`  | 输出 32 位 | Load/Store 有效地址                             |
| `daccess_rvalid`      | 输入 1 位  | Load 返回数据有效                               |
| `daccess_rdata[31:0]` | 输入 32 位 | Load 返回的对齐 32 位字                         |
| `daccess_wen[3:0]`    | 输出 4 位  | Store 的四字节写使能                            |
| `daccess_wdata[31:0]` | 输出 32 位 | Store 数据，已按地址低位移到正确 lane           |
| `daccess_wresp`       | 输入 1 位  | Store 完成应答                                  |

`daccess_addr`、`daccess_wdata` 在对应使能为 0 时可以是无关值；解释波形时必须和 `daccess_ren/daccess_wen` 一起看。

### 4.2 PC 与取指信号

| 信号                | 含义                                                                                       |
| ------------------- | ------------------------------------------------------------------------------------------ |
| `pc`              | 当前正在执行的指令地址                                                                     |
| `npc`             | 组合逻辑计算出的下一条指令地址                                                             |
| `pc4`             | `pc+4`，顺序执行地址，也是 JAL/JALR 链接地址                                             |
| `inst`            | 核心内部使用的指令；`ifetch_valid=0` 时强制为 `0x00000013`（`addi x0,x0,0`，即 NOP） |
| `rst_r`           | 上一时钟沿采样到的复位值                                                                   |
| `first_req`       | `rst_r & !cpu_rst`，检测复位刚撤销，用于发出第一次取指                                   |
| `ifetch_req`      | `first_req                                                                                 |
| `inst_finished_r` | `inst_finished` 的一拍寄存，用于把完成事件转换为下一次取指请求                           |

PC 模块的 `fetch` 直接连接 `inst_finished`：普通指令在有效指令到达时推进 PC，访存和乘除法则等完成条件满足后再推进。

### 4.3 Controller 控制信号

| 信号           | 位宽 | 含义                                                 |
| -------------- | ---: | ---------------------------------------------------- |
| `npc_op`     |    2 | NPC 选择：顺序、JALR、条件分支或 JAL                 |
| `rf_wsel`    |    2 | 写回数据选择：ALU、RAM、PC+4 或扩展立即数            |
| `sext_op`    |    3 | 立即数格式选择：I/S/B/U/J                            |
| `alu_op`     |    5 | ALU 运算或分支比较类型                               |
| `alua_sel`   |    1 | ALU A 选 RS1 或 PC                                   |
| `alub_sel`   |    1 | ALU B 选 RS2 或扩展立即数                            |
| `ram_rop`    |    3 | Load 宽度及符号/零扩展类型                           |
| `ram_rop_r`  |    3 | 保存的 Load 类型，等待存储器返回期间继续供 MEXT 使用 |
| `ram_wop`    |    4 | Store 类型：不写、字节、半字或字                     |
| `is_mul`     |    1 | 当前译码结果是乘法类指令                             |
| `is_div`     |    1 | 当前译码结果是除法或余数类指令                       |
| `is_mul_div` |    1 | `is_mul                                              |

### 4.4 寄存器堆与写回信号

| 信号        | 含义                                                    |
| ----------- | ------------------------------------------------------- |
| `rf_rd1`  | RF 第一个读口数据，即 RS1 的值                          |
| `rf_rd2`  | RF 第二个读口数据，即 RS2 的值，也是 Store 写数据       |
| `rf_rd3`  | 当前 RTL 中未连接、未使用的遗留信号，对功能无影响       |
| `rf_we`   | Controller 给出的“该指令需要写 rd”意图                |
| `rf_we1`  | 真正送到 RF 的时序写使能；只有它为 1 才发生写回         |
| `rf_wR`   | 最终写寄存器编号                                        |
| `rf_wR_r` | 为 Load/乘除法保存的 rd，因为返回结果时输入指令已经无效 |
| `rf_wD`   | 最终写回数据                                            |

`rf_we1` 的三个来源：

```text
Load：      ld_st_flag & daccess_rvalid
乘除法：    mul_div_flag & !mul_div_busy
普通指令：  ifetch_valid & rf_we & !is_ld_st & !is_mul_div
```

`rf_wR` 在多周期指令期间选择保存的 `rf_wR_r`，否则直接选择 `inst[11:7]`。`rf_wD` 的组合多路器选择 ALU、PC+4、扩展立即数或 `ram_ext`；Load 期间由 `ld_st_flag` 强制选择 RAM 数据。

### 4.5 ALU 数据与分支信号

| 信号             | 含义                                           |
| ---------------- | ---------------------------------------------- |
| `ext`          | SEXT 生成的 32 位立即数                        |
| `alu_a`        | `alua_sel=1` 时为 PC，否则为 `rf_rd1`      |
| `alu_b`        | `alub_sel=1` 时为 `ext`，否则为 `rf_rd2` |
| `alu_c`        | ALU 32 位运算结果；也用作 Load/Store/JALR 地址 |
| `alu_c_r`      | 保存的访存有效地址，用于响应周期选择字节/半字  |
| `br`           | ALU 输出的分支条件是否成立                     |
| `mul_div_busy` | 任一乘法器或除法器仍在迭代                     |

### 4.6 访存状态和接口寄存

| 信号                | 含义                                                    |
| ------------------- | ------------------------------------------------------- |
| `da_ren/da_addr`  | MREQ 组合生成的读请求和地址                             |
| `da_wen/da_wdata` | MREQ 组合生成的写使能和对齐数据                         |
| `ram_ext`         | MEXT 选择并扩展后的 Load 最终数据                       |
| `is_ld_st`        | `ram_rop` 或 `ram_wop` 非空，说明当前译码为访存指令 |
| `ld_st_flag`      | 跨周期保存“正在进行访存”状态                          |
| `ld_st_done`      | `daccess_rvalid                                         |

`daccess_*` 在时钟沿锁存 `da_*`，使核心对外形成稳定的总线请求。开始访存时还保存 `alu_c_r`、`ram_rop_r` 和 `rf_wR_r`。

### 4.7 乘除法状态

| 信号             | 含义                               |
| ---------------- | ---------------------------------- |
| `mul_div_flag` | 表示核心正在等待一条乘除法指令完成 |
| `mul_div_busy` | ALU 内实际迭代单元的忙信号         |
| `rf_wR_r`      | 保存该乘除法指令的 rd              |

开始时 `is_mul_div=1`，核心置位 `mul_div_flag` 并保存 rd。busy 期间 PC 保持；busy 变为 0 后，`rf_we1` 和 `inst_finished` 同时有效，结果写回并推进 PC。

### 4.8 指令完成逻辑

```text
访存完成：  ld_st_flag & ld_st_done
乘除完成：  mul_div_flag & !mul_div_busy
普通完成：  ifetch_valid & !is_ld_st & !is_mul_div
```

注意 Store 虽然不写寄存器，但仍必须等待 `daccess_wresp` 才算完成。分支不写寄存器，却属于普通单周期指令，因此在 `ifetch_valid` 当拍根据 `npc` 更新 PC。

### 4.9 Trace 调试接口

| 信号                | 含义                            |
| ------------------- | ------------------------------- |
| `debug_wb_pc`     | 写回/提交指令的 PC              |
| `debug_wb_rf_we`  | 提交时是否写 RF，等于`rf_we1` |
| `debug_wb_rf_wR`  | 写回寄存器号                    |
| `debug_wb_rf_wD`  | 写回值                          |
| `debug_mem_pc`    | 访存指令 PC                     |
| `debug_mem_we`    | 对外数据写使能                  |
| `debug_mem_waddr` | 写内存地址                      |
| `debug_mem_wdata` | 写内存数据                      |

这些信号只用于仿真 Trace 和差分测试，不参与 CPU 功能计算。

## 五、`Controller`：指令译码器

文件：[`Controller.v`](../miniRV_basic/src/rtl/Controller.v)

### 输入接口

| 端口            | 含义                                                      |
| --------------- | --------------------------------------------------------- |
| `opcode[6:0]` | 指令主操作码`inst[6:0]`，首先确定大类                   |
| `funct3[2:0]` | `inst[14:12]`，区分同一大类中的具体运算、宽度或比较方式 |
| `funct7[6:0]` | `inst[31:25]`，区分 ADD/SUB、SRL/SRA 和 M 扩展等        |

### 输出接口

| 端口                  | 功能                                 |
| --------------------- | ------------------------------------ |
| `npc_op`            | 决定下一 PC 计算方式                 |
| `sext_op`           | 决定立即数字段如何重排与扩展         |
| `alua_sel/alub_sel` | 控制两个 ALU 输入多路器              |
| `alu_op`            | 指定算术、逻辑、移位、比较或乘除运算 |
| `is_mul/is_div`     | 通知核心进入乘法或除法多周期流程     |
| `ram_r_op`          | 指定 Load 数据宽度以及符号/零扩展    |
| `ram_w_op`          | 指定 Store 写入字节数                |
| `rf_we`             | 指令是否有 rd 写回                   |
| `rf_wsel`           | rd 写回值来自哪里                    |

### 内部功能块

1. 每个 `wire ADDI/.../REMU` 都是一个指令匹配条件，不是存储状态。
2. `load_inst/store_inst/branch_inst/reg_alu_inst/imm_alu_inst/mul_div_inst` 把具体指令归类，减少后续控制表达式重复。
3. `npc_op` 按 JALR、分支、JAL、顺序执行的优先级选择。
4. `rf_we` 汇总所有会写 rd 的指令；Store 和条件分支不在其中。
5. `rf_wsel` 在 RAM、PC+4、扩展立即数、ALU 四个来源中选择。
6. `sext_op` 在 S/B/U/J/I 型立即数之间选择。
7. `alu_op` 是优先条件链，若没有匹配特殊运算则默认 ADD，用于 ADDI、地址计算和 AUIPC。
8. `alua_sel` 只有 AUIPC 选 PC，其余选 RS1；`alub_sel` 为立即数运算、地址计算和 JALR 选择 EXT。
9. `ram_r_op/ram_w_op` 编码访存宽度；`is_mul/is_div` 汇总多周期指令。

## 六、`PC` 与 `NPC`

### `PC` 模块

文件：[`PC.v`](../miniRV_basic/src/rtl/PC.v)

| 端口          | 含义                         |
| ------------- | ---------------------------- |
| `clk/rst`   | 时钟和高电平异步复位         |
| `npc[31:0]` | 待写入的下一 PC              |
| `fetch`     | 当前指令已完成，允许 PC 更新 |
| `pc[31:0]`  | 保存的当前 PC                |

复位时 `pc=0`；否则仅在 `fetch=1` 的时钟沿执行 `pc<=npc`，`fetch=0` 时保持原 PC。

### `NPC` 模块

文件：[`NPC.v`](../miniRV_basic/src/rtl/NPC.v)

| 端口          | 含义                   |
| ------------- | ---------------------- |
| `op[1:0]`   | 下一 PC 选择控制       |
| `pc`        | 当前 PC                |
| `offset`    | SEXT 生成的 B/J 型偏移 |
| `jalr_addr` | ALU 计算的`rs1+imm`  |
| `br`        | 条件分支是否成立       |
| `npc`       | 最终下一 PC            |
| `pc4`       | 当前 PC 加 4           |

四种功能：`PC4` 取 `pc+4`；`JALR` 取 `{jalr_addr[31:1],1'b0}`；`BRA` 根据 `br` 选择 `pc+offset` 或 `pc+4`；`JMP` 直接取 `pc+offset`。

## 七、`RF`：整数寄存器堆

文件：[`RF.v`](../miniRV_basic/src/rtl/RF.v)

| 端口              | 含义             |
| ----------------- | ---------------- |
| `clk`           | 写回时钟         |
| `rR1/rR2[4:0]`  | 两个源寄存器编号 |
| `rD1/rD2[31:0]` | 两个组合读口数据 |
| `we`            | 写使能           |
| `wR[4:0]`       | 目标寄存器编号   |
| `wD[31:0]`      | 写回数据         |

内部只声明 `regs[1:31]`，没有为 x0 分配可写存储。读地址为 0 时直接返回 0；写回还检查 `wR!=0`，所以 x0 永远为 0。读是组合逻辑，写在时钟上升沿发生。

## 八、`SEXT`：立即数生成器

文件：[`SEXT.v`](../miniRV_basic/src/rtl/SEXT.v)

| 端口          | 含义                           |
| ------------- | ------------------------------ |
| `op[2:0]`   | 立即数格式控制                 |
| `imm[31:7]` | 指令中可能包含立即数的高 25 位 |
| `ext[31:0]` | 重排并扩展后的 32 位立即数     |

| 格式      | 拼接方式                                         | 用途                          |
| --------- | ------------------------------------------------ | ----------------------------- |
| `EXT_I` | 符号扩展`inst[31:20]`                          | I 型运算、Load、JALR          |
| `EXT_S` | 拼接`inst[31:25]` 与 `inst[11:7]` 后符号扩展 | Store                         |
| `EXT_B` | 重排 B 型字段并补最低位 0                        | 条件分支，偏移一定为 2 的倍数 |
| `EXT_U` | `inst[31:12]` 后补 12 个 0                     | LUI、AUIPC                    |
| `EXT_J` | 重排 J 型字段并补最低位 0                        | JAL                           |

## 九、`ALU`：执行、比较与乘除法封装

文件：[`ALU.v`](../miniRV_basic/src/rtl/ALU.v)

### 外部端口

| 端口          | 含义                                               |
| ------------- | -------------------------------------------------- |
| `rst/clk`   | 乘除法内部状态的复位和时钟；普通组合运算不等待时钟 |
| `op[4:0]`   | ALU 操作码                                         |
| `a/b[31:0]` | 两个操作数                                         |
| `c[31:0]`   | 普通运算或最终乘除结果                             |
| `br`        | 分支比较结果                                       |
| `busy`      | 任一乘除迭代器正忙                                 |

### 普通组合运算

| 操作           | 功能                                    |
| -------------- | --------------------------------------- |
| `ADD/SUB`    | 32 位加减，保留低 32 位                 |
| `OR/XOR/AND` | 逐位逻辑运算                            |
| `SLL`        | `a << b[4:0]`                         |
| `SRL`        | 无符号逻辑右移，高位补 0                |
| `SRA`        | `$signed(a) >>> b[4:0]`，高位补符号位 |
| `SLT`        | 有符号小于则输出 1                      |
| `SLTU`       | 无符号小于则输出 1                      |

分支 `br` 单独由第二个组合块生成：`EQ/NE` 比较相等，`LT/GE` 使用有符号比较，`LTU/GEU` 使用无符号比较。

### 乘除法内部信号

| 信号                       | 含义                                                |
| -------------------------- | --------------------------------------------------- |
| `mul_flag`               | 启动有符号`MUL/MULH` 所用乘法器                   |
| `mulu_flag`              | 启动`MULHU` 无符号乘法器                          |

| `mul_busy/mulu_busy`     | 对应乘法器忙                                        |

## 十一、`divider`：参数化恢复除法器

文件：[`divider.v`](../miniRV_basic/src/rtl/divider.v)

### 参数和端口

| 名称                   | 含义                       |
| ---------------------- | -------------------------- |
| `parameter WIDTH=32` | 被除数、除数、商和余数位宽 |
| `x`                  | 被除数                     |
| `y`                  | 除数                       |
| `start`              | 在 IDLE 启动运算           |
| `z`                  | 商 quotient                |
| `r`                  | 余数 remainder             |
| `busy`               | 正在逐位计算               |

### 内部状态和数据

| 名称          | 含义                                                    |
| ------------- | ------------------------------------------------------- |
| `R`         | 当前部分余数                                            |
| `Q`         | 逐位生成的商                                            |

| `dividend`  | 锁存的被除数                                            |

## 十二、`MREQ`：访存请求与写数据对齐

文件：[`MREQ.v`](../miniRV_basic/src/rtl/MREQ.v)

| 端口            | 含义                           |
| --------------- | ------------------------------ |
| `ram_addr`    | ALU 算出的字节地址             |
| `ram_rop`     | Load 类型                      |
| `da_ren[3:0]` | 对外读请求                     |
| `da_addr`     | 对外地址，直接等于`ram_addr` |
| `ram_wop`     | Store 类型                     |
| `ram_wdata`   | RF 的 RS2 数据                 |
| `da_wen[3:0]` | 对外字节 lane 写使能           |
| `da_wdata`    | 移到目标 lane 后的写数据       |

`offset=ram_addr[1:0]` 表示目标字节在 32 位字内的位置。

- `sb`：`0001 << offset`，并把低 8 位左移 `offset*8` 位。
- `sh`：只允许 offset 为 0 或 2，写使能分别为 `0011/1100`。
- `sw`：只允许 offset 为 0，写使能为 `1111`。
- 字节读允许任意 offset；半字读要求 `offset[0]=0`；字读要求 offset=0。

本设计对未对齐半字/字访问的处理是不给出有效请求，而不是实现跨两个字的访问。

## 十六、接口握手的完整时间顺序

### 普通单周期指令

```text
ifetch_req → 下一拍 ifetch_valid+ifetch_inst
            → 当拍译码、ALU、可选 RF 写回、inst_finished
            → PC 在时钟沿更新，随后产生下一次取指请求
```

### Load

```text
有效指令 → ALU 算地址 → MREQ 生成 da_ren
         → 时钟沿输出 daccess_ren，并保存 rd/地址/读类型
         → 存储器返回 daccess_rvalid+daccess_rdata
         → MEXT 选择并扩展 → rf_we1 写回 → PC 更新
```

### Store

```text
有效指令 → ALU 算地址，RS2 作为写数据 → MREQ 对齐并生成 da_wen
         → 时钟沿输出 daccess_wen/daccess_wdata
         → 存储器返回 daccess_wresp → PC 更新，不写 RF
```

### 乘除法

```text
有效指令 → start，保存 op/操作数/rd → busy=1
         → 多拍迭代，PC 保持
         → busy=0，结果选择有效 → rf_we1 写回 → PC 更新
```

## 十七、答辩中常见的接口追问

1. **为什么有 `rf_we` 还需要 `rf_we1`？** `rf_we` 只说明指令语义需要写回；`rf_we1` 保证数据真正有效的那一拍才写，尤其适用于 Load 和乘除法。
2. **为什么要保存 `rf_wR_r`？** 多周期结果返回时原 `inst` 已因 `ifetch_valid=0` 被替换为 NOP，不保存就会丢失 rd。
3. **为什么要保存 `alu_c_r` 和 `ram_rop_r`？** Load 返回时仍要知道原地址低位以及做字节、半字、符号还是零扩展。
4. **为什么 Store 数据不用 `alu_b`？** ALU B 用来接立即数计算地址；真正写入内存的数据从 RF 第二读口 `rf_rd2` 独立送往 MREQ。
5. **为什么分支的 ALU 结果 `c` 不重要？** 分支使用的是独立输出 `br`，真正的目标地址由 NPC 用 `pc+ext` 计算。
6. **为什么 `inst` 会在波形中变成 `0x13`？** 返回指令只保证 `ifetch_valid` 那一拍有效，核心主动用 NOP 屏蔽无效时段，避免错误译码。
7. **为什么地址送给 BRAM时用 `[31:2]`？** CPU 地址以字节为单位，而 BRAM 每个单元是 32 位字，低两位表示字内 byte offset。
8. **`busy` 和 `mul_div_flag` 有什么区别？** busy 是迭代器当前是否工作；flag 是核心是否仍有一条乘除法指令等待提交，二者共同决定完成和写回时刻。

## 十八、课程数据通路表中的“输入信号”逐列解释

课程表：[`miniRV_AB组_数据通路表+控制信号表.xlsx`](../course-materials/miniRV/miniRV_AB组_数据通路表+控制信号表.xlsx)

数据通路表的一行代表一条指令。某一格写 `RF.rD1`、`ALU.C` 等，表示该部件的对应输入端应连接到这个来源；它描述的是“数据从哪里来”，不表示一个固定数值。表中的 `-` 表示这条指令不使用该输入，属于无关项，不一定在实际组合电路波形中为 0。

### 18.1 PC 部件

| 表格输入 | 来源        | 含义                                                                                               |
| -------- | ----------- | -------------------------------------------------------------------------------------------------- |
| `npc`  | `NPC.npc` | 下一条指令地址。只有当前指令完成、`fetch/inst_finished=1` 时，PC 才在时钟沿把它保存为新的 `pc` |

所有指令都需要给 PC 准备下一地址，但访存和乘除法没有完成时 PC 不会立即采用它。

### 18.2 NPC 部件

| 表格输入   | 来源                                               | 含义                                                                                          |
| ---------- | -------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| `pc`     | `PC.pc`                                          | 当前指令地址，是计算`pc+4`、分支目标和 JAL 目标的基准                                       |
| `offset` | `SEXT.ext` 或 JALR 表中的 `{ALU.C[31:1],1'b0}` | B/J 型时是相对 PC 的立即数偏移；JALR 实际使用独立的`jalr_addr=ALU.C`，并在 NPC 内清零最低位 |
| `br`     | `ALU.br`                                         | 条件分支是否成立；只对 BEQ/BNE/BLT/BGE/BLTU/BGEU 有意义                                       |

`npc_op` 决定 NPC 如何使用这些输入。普通指令即使 `offset/br` 波形有某个值，只要 `npc_op=PC4`，它们就不会影响下一 PC。

### 18.3 RF 寄存器堆

| 表格输入 | 来源                                                 | 含义                                                       |
| -------- | ---------------------------------------------------- | ---------------------------------------------------------- |
| `rR1`  | `IN.inst[19:15]`                                   | RS1 编号，选择第一个源寄存器                               |
| `rR2`  | `IN.inst[24:20]`                                   | RS2 编号，选择第二个源寄存器；Store 的待写数据也从这里读取 |
| `wR`   | `IN.inst[11:7]` 或保存的 `rf_wR_r`               | rd 编号；Load/乘除法跨周期时使用事先保存的编号             |
| `wD`   | `ALU.C`、`SEXT.ext`、`MEXT.ext` 或 `NPC.pc4` | 写入 rd 的数据，具体来源由`rf_wsel` 和访存状态选择       |

表格里的 `IN` 表示 CPU 核心的输入/指令输入一侧。当前 RTL 中对应的是内部有效指令 `inst`；只有 `ifetch_valid=1` 时才能把 `ifetch_inst` 当成有效指令解释。

### 18.4 SEXT 立即数生成器

| 表格输入 | 来源                           | 含义                                                                                     |
| -------- | ------------------------------ | ---------------------------------------------------------------------------------------- |
| `imm`  | `IN.inst[31:7]` 的不同子字段 | 原始立即数字段。SEXT 根据`sext_op` 决定按 I/S/B/U/J 哪一种格式重排、补最低位并符号扩展 |

表格可能写成：

- `IN.inst[31:20]`：I 型 12 位立即数；
- `IN.inst[31:25|11:7]`：S 型立即数分成两段；
- `IN.inst[31|7|30:25|11:8]`：B 型立即数重排；
- `IN.inst[31:12]`：U 型高 20 位；
- `IN.inst[31|19:12|20|30:21]`：J 型立即数重排。

竖线 `|` 在表格中表示字段拼接顺序，不是逻辑或运算。

### 18.5 ALU 执行单元

| 表格输入 | 来源                       | 含义                                                                     |
| -------- | -------------------------- | ------------------------------------------------------------------------ |
| `A`    | `RF.rD1` 或 `PC.pc`    | 第一个操作数；AUIPC 选 PC，其他当前支持指令通常选 RS1                    |
| `B`    | `RF.rD2` 或 `SEXT.ext` | 第二个操作数；R 型、分支和乘除法选 RS2，立即数运算与地址计算选扩展立即数 |

`alua_sel` 和 `alub_sel` 控制这两个来源，`alu_op` 决定 ALU 对 A、B 做加、减、逻辑、移位、比较还是乘除运算。

### 18.6 MREQ 访存请求生成器

| 表格输入      | 来源       | 含义                                                                           |
| ------------- | ---------- | ------------------------------------------------------------------------------ |
| `ram_addr`  | `ALU.C`  | `rs1+立即数` 得到的 Load/Store 有效字节地址                                  |
| `ram_wdata` | `RF.rD2` | Store 要写入的数据，即 RS2；SB 只用低 8 位，SH 只用低 16 位，SW 使用全部 32 位 |

### 18.7 MEXT 读数据扩展器

| 表格输入      | 来源                                                  | 含义                         |
| ------------- | ----------------------------------------------------- | ---------------------------- |
| `din`       | `IN.daccess_rdata`                                  | 数据存储器返回的完整 32 位字 |
| `byte_offs` | `ALU.C[1:0]`，RTL 中实际使用保存的 `alu_c_r[1:0]` | 目标字节在返回字内的位置     |

MEXT 根据 `byte_offs` 把目标字节/半字移到最低位，再按保存的 `ram_rop_r` 做符号扩展或零扩展。表格写 `ALU.C[1:0]` 是数据来源的抽象描述；RTL 保存它是因为响应比地址计算晚若干拍。

## 十九、课程控制信号表逐列解释

控制信号表中的前三列 `opcode/funct3/funct7` 是译码依据；后面的列是 Controller 根据译码结果产生的控制输出。

### 19.1 译码输入列

| 列名       | 指令位段        | 作用                                                |
| ---------- | --------------- | --------------------------------------------------- |
| `opcode` | `inst[6:0]`   | 判断 R/I/Load/Store/Branch/JAL/JALR/U 型等主类别    |
| `funct3` | `inst[14:12]` | 区分加法/逻辑/比较、访存宽度、分支条件和 M 扩展运算 |
| `funct7` | `inst[31:25]` | 区分 ADD/SUB、SRL/SRA、普通 R 型/M 扩展等           |

表中的 `-` 表示识别该指令时不需要检查该字段。例如 AUIPC 只靠 opcode 已经唯一，不要求 funct3/funct7。

### 19.2 `npc_op`：下一 PC 操作选择

| 表格取值   | RTL 宏/编码     | 功能                       |
| ---------- | --------------- | -------------------------- |
| `PC4`    | `NPC_PC4=00`  | `npc=pc+4`               |
| `JALR`   | `NPC_JALR=01` | `npc={alu_c[31:1],1'b0}` |
| `BRANCH` | `NPC_BRA=10`  | `br?pc+ext:pc+4`         |
| `JMP`    | `NPC_JMP=11`  | `npc=pc+ext`，用于 JAL   |

### 19.3 `rf_we`：寄存器堆写意图

| 取值  | 含义                                                       |
| ----- | ---------------------------------------------------------- |
| `1` | 该指令语义需要写 rd，例如 ALU、Load、LUI、跳转链接和乘除法 |
| `0` | 不写 rd，例如 Store 和条件分支                             |

它不是最终写脉冲。最终波形必须看 `rf_we1`；Load 和乘除法会延迟到结果有效时才真正写。

### 19.4 `rf_wsel`：写回数据来源

| 表格取值   | RTL 宏/编码 | 写回数据                                |
| ---------- | ----------- | --------------------------------------- |
| `WB_ALU` | `00`      | `ALU.C`，用于普通 ALU、AUIPC、乘除法  |
| `WB_RAM` | `01`      | `MEXT.ext/ram_ext`，用于 Load         |
| `WB_PC4` | `10`      | `NPC.pc4/pc4`，用于 JAL/JALR 返回地址 |
| `WB_EXT` | `11`      | `SEXT.ext`，用于 LUI                  |
| `-`      | 无关        | `rf_we=0` 时不会写 RF，所以来源无意义 |

### 19.5 `sext_op`：立即数格式

| 表格取值   | RTL 宏/编码   | 功能                                  |
| ---------- | ------------- | ------------------------------------- |
| `I_TYPE` | `EXT_I=000` | 符号扩展`inst[31:20]`               |
| `S_TYPE` | `EXT_S=001` | 拼接 Store 立即数并符号扩展           |
| `B_TYPE` | `EXT_B=010` | 重排分支立即数、补最低位 0 并符号扩展 |
| `U_TYPE` | `EXT_U=011` | 高 20 位后补 12 个 0                  |
| `J_TYPE` | `EXT_J=100` | 重排 JAL 偏移并补最低位 0             |
| `-`      | 无关          | 该指令不使用立即数，例如多数 R 型指令 |

### 19.6 `alu_op`：ALU 功能选择

| 表格取值                | 功能                                                   |
| ----------------------- | ------------------------------------------------------ |
| `ADD/SUB`             | 加法或减法；ADD 还用于有效地址、AUIPC 和 JALR 目标计算 |
| `OR/XOR/AND`          | 逐位逻辑运算                                           |
| `SLL/SRL/SRA`         | 左移、逻辑右移、算术右移                               |
| `SLT/SLTU`            | 输出有符号/无符号小于比较的 0 或 1                     |
| `EQ/NE/LT/GE/LTU/GEU` | 产生分支输出`br`，不写比较值                         |
| `MUL/MULH/MULHU`      | 乘积低位、有符号积高位、无符号积高位                   |
| `DIV/DIVU`            | 有符号/无符号商                                        |
| `REM/REMU`            | 有符号/无符号余数                                      |

### 19.7 `alua_sel` 与 `alub_sel`：ALU 输入 MUX

| 信号/表格取值        | RTL 编码 | 选择的数据                  |
| -------------------- | -------- | --------------------------- |
| `alua_sel=SEL_RS1` | `0`    | `A=RF.rD1`                |
| `alua_sel=SEL_PC`  | `1`    | `A=PC.pc`，当前用于 AUIPC |
| `alub_sel=SEL_RS2` | `0`    | `B=RF.rD2`                |
| `alub_sel=SEL_EXT` | `1`    | `B=SEXT.ext`              |

有些旧版示例表只单独列出 `alub_sel`；完整表还应看 `alua_sel`。选择信号只决定来源，真正进行哪种运算仍由 `alu_op` 决定。

### 19.8 `ram_rop`：Load 读回处理

| 表格取值        | RTL 宏/编码        | 功能               |
| --------------- | ------------------ | ------------------ |
| `0`           | `RAM_EXT_N=000`  | 不读数据存储器     |
| `WORD_EXT`    | `RAM_EXT_W=001`  | 读取完整 32 位字   |
| `BYTE_EXT`    | `RAM_EXT_B=010`  | 读取字节并符号扩展 |
| `BYTE_UNSIGN` | `RAM_EXT_BU=011` | 读取字节并零扩展   |
| `HALF_EXT`    | `RAM_EXT_H=100`  | 读取半字并符号扩展 |
| `HALF_UNSIGN` | `RAM_EXT_HU=101` | 读取半字并零扩展   |

`ram_rop` 同时表示“是否是 Load”和“返回数据如何扩展”。

### 19.9 `ram_wop`：Store 写入宽度

| 表格取值       | RTL 宏/编码       | 功能                                   |
| -------------- | ----------------- | -------------------------------------- |
| `0`          | `RAM_WE_N=0000` | 不写数据存储器                         |
| `BYTE_WRITE` | `RAM_WE_B=0001` | 写 RS2 低 8 位；MREQ 再按地址移动 lane |
| `HALF_WRITE` | `RAM_WE_H=0011` | 写 RS2 低 16 位                        |
| `WORD_WRITE` | `RAM_WE_W=1111` | 写完整 32 位字                         |

表里的 `0001/0011/1111` 是写宽度的基础编码。最终对外 `daccess_wen` 还会根据地址低位变化，例如 SB 可能输出 `0010/0100/1000`，SH 可能输出 `1100`。

## 二十、如何现场读表格的一整行

以 `lb rd, imm(rs1)` 为例：

1. `rR1=inst[19:15]`，RF 读出基址；`imm=inst[31:20]`，SEXT 做 I 型符号扩展。
2. `alua_sel=SEL_RS1`、`alub_sel=SEL_EXT`、`alu_op=ADD`，所以 ALU.C 是有效地址。
3. `ram_addr=ALU.C`、`ram_rop=BYTE_EXT`，MREQ 发读请求，MEXT 按地址低位选择目标字节并符号扩展。
4. `rf_we=1`、`rf_wsel=WB_RAM`，返回有效时把 `MEXT.ext` 写入 `inst[11:7]` 指定的 rd。
5. `npc_op=PC4`，访存应答完成后 PC 才更新为 `pc+4`。

读任何一行都可以使用同样顺序：先看字段与 RF/SEXT 输入，再看 ALU 两个 MUX 和操作，接着看访存，最后看写回和下一 PC。
